# ZiFi FTP Server для Wild Commander

`ZiFi FTP Server` — FTP-сервер в формате плагина `.WMF` для ZX Evolution с TS-Config, Wild Commander и сетевым интерфейсом ZiFi. Он позволяет работать с SD-картой компьютера по локальной сети: просматривать каталоги, скачивать и загружать файлы, создавать папки, переименовывать и удалять объекты.

Корень FTP `/` совпадает с корнем выбранной SD-карты. Все файловые операции выполняются встроенным FAT-драйвером Wild Commander; плагин не обращается к SD-контроллеру напрямую.

## Скачать готовую версию

Готовый файл `ZIFIFTP.WMF` публикуется в разделе [Releases](https://github.com/andrewinsidelazarev/ZiFi-FTP-Server/releases). Для обычной установки исходники и компилятор не нужны.

## Возможности

- запуск из меню плагинов Wild Commander по `F10`;
- чтение настроек Wi-Fi и FTP из `/zifi/zifi.ini`;
- подключение ZiFi к Wi-Fi и ожидание DHCP-адреса;
- показ состояния сервера и адреса вида `ftp://192.168.x.x:21`;
- доступ на чтение и запись ко всему выбранному тому SD-карты;
- просмотр каталогов и скачивание файлов через `LIST`, `NLST` и `RETR`;
- загрузка файлов на SD через `STOR`;
- создание, удаление и переименование файлов и каталогов;
- активный FTP через `PORT` и `EPRT`;
- пакетное чтение очереди ESP командой Z80 `INIR`;
- завершение работы сервера по `Esc`.

## Требования

- ZX Evolution с TS-Config;
- Wild Commander с поддерживаемым API плагинов;
- ZiFi с AT-прошивкой ESP;
- SD-карта, доступная Wild Commander;
- Wi-Fi-сеть, к которой могут одновременно подключиться ZiFi и компьютер с FTP-клиентом;
- FTP-клиент с поддержкой активного режима, например FileZilla.

> [!WARNING]
> После входа FTP-клиент получает право изменять весь выбранный том: создавать, заменять, переименовывать и удалять файлы. Перед первым испытанием сделайте резервную копию SD-карты.

## Быстрый запуск

1. Скачайте `ZIFIFTP.WMF` со страницы последнего релиза.
2. Скопируйте `ZIFIFTP.WMF` в каталог `/WC` загрузочного устройства Wild Commander.
3. Добавьте строку `ZIFIFTP.WMF` в секцию `[PLUGINS]` файла `/WC/wc.ini`.
4. Создайте каталог `/zifi` в корне SD-карты.
5. Создайте файл `/zifi/zifi.ini` и укажите в нём имя Wi-Fi-сети и пароль.
6. Запустите Wild Commander, нажмите `F10` и выберите `ZiFi FTP Server`.
7. Дождитесь состояния `Listening` и запишите показанный IP-адрес.
8. Подключитесь к этому адресу FTP-клиентом в активном режиме.

## Настройка `zifi.ini`

Файл конфигурации должен находиться по пути:

```text
/zifi/zifi.ini
```

Минимальная конфигурация:

```ini
SSID: Имя сети
password: Пароль Wi-Fi
```

Полная конфигурация:

```ini
SSID: Имя сети
password: Пароль Wi-Fi
ftp_user: zx
ftp_password: zx
ftp_port: 21
ftp_timeout: 600
```

| Параметр | Обязательный | Значение по умолчанию | Назначение |
| --- | --- | --- | --- |
| `SSID` | да | — | Имя Wi-Fi-сети |
| `password` | да | — | Пароль Wi-Fi |
| `ftp_user` | нет | `zx` | Имя пользователя FTP |
| `ftp_password` | нет | `zx` | Пароль FTP |
| `ftp_port` | нет | `21` | Порт управляющего FTP-соединения |
| `ftp_timeout` | нет | `600` | Тайм-аут операций ZiFi в секундах |

Имена ключей не зависят от регистра. Значения, включая `ftp_port` и `ftp_timeout`, разрешается записывать с двойными кавычками или без них. Поддерживаются окончания строк CR, LF и CRLF, а также UTF-8 как без BOM, так и с BOM `EF BB BF`. Все параметры должны полностью помещаться в первые 511 байт файла.

Готовый шаблон находится в [zifi.ini.example](zifi.ini.example).

### Как выбирается SD-карта

Плагин ищет `/zifi/zifi.ini` в следующем порядке:

1. устройство активной панели Wild Commander;
2. SD1 Z-Controller;
3. SD2 Z-Controller.

FTP открывает корень именно того тома, на котором найден файл конфигурации. Для SD NeoGS каталог `/zifi` должен быть доступен на активной панели в момент запуска плагина.

## Подробная настройка FileZilla

Сервер поддерживает только активный FTP. Пассивные команды `PASV` и `EPSV` намеренно недоступны, поскольку управляющий канал уже занимает единственный прослушивающий сервер ESP.

### 1. Настройте активные порты

Откройте в FileZilla:

```text
Правка → Настройки → Соединение → FTP → Активный режим
```

Включите ограничение локальных портов и задайте, например:

```text
нижний порт: 50000
верхний порт: 50049
```

Разрешите входящие TCP-соединения на этот диапазон в брандмауэре компьютера. Правило желательно ограничить профилем локальной сети и IP-адресом ZiFi.

Если ZiFi и компьютер находятся в одной локальной сети, FileZilla должен передавать серверу локальный IPv4-адрес компьютера, например `192.168.1.50`, а не внешний адрес маршрутизатора.

### 2. Создайте профиль сервера

Откройте `Файл → Менеджер сайтов`, создайте новый сайт и укажите:

| Поле | Значение |
| --- | --- |
| Протокол | `FTP — протокол передачи файлов` |
| Хост | IP, показанный плагином, например `192.168.1.105` |
| Порт | `21` или значение `ftp_port` |
| Шифрование | обычный FTP без TLS |
| Тип входа | обычный |
| Пользователь | значение `ftp_user`, по умолчанию `zx` |
| Пароль | значение `ftp_password`, по умолчанию `zx` |

На вкладке настроек передачи:

1. выберите активный режим;
2. включите ограничение количества одновременных соединений;
3. установите максимум `1`.

### 3. Подключитесь и проверьте операции

После подключения правая панель FileZilla должна показать корень SD-карты. Для безопасной проверки:

1. создайте каталог с тестовым именем;
2. загрузите в него небольшой файл;
3. скачайте файл обратно и сравните содержимое;
4. переименуйте файл;
5. удалите файл;
6. удалите пустой каталог.

Загрузка выполняется обычным перетаскиванием файла с левой панели FileZilla на правую. Скачивание выполняется в обратном направлении.

## Как работает активный FTP

FTP использует два TCP-соединения:

1. компьютер открывает управляющее соединение к ZiFi на порт `21`;
2. перед выдачей списка или передачей файла клиент сообщает свой адрес командой `PORT` или `EPRT`;
3. ZiFi открывает обратное data-соединение к указанному порту компьютера;
4. после завершения операции data-соединение закрывается, а управляющий канал остаётся активным.

Поэтому успешный вход ещё не гарантирует передачу файлов. Если каталог не открывается после авторизации, прежде всего проверяйте активный режим, локальный IP и входящее правило брандмауэра.

## Поддерживаемые FTP-команды

Поддерживаются:

```text
USER PASS QUIT SYST FEAT NOOP
PWD XPWD CWD XCWD CDUP TYPE
PORT EPRT LIST NLST RETR STOR
SIZE MDTM DELE MKD XMKD RMD XRMD
RNFR RNTO OPTS CLNT MODE STRU STAT ABOR
```

Не поддерживаются:

- `PASV` и `EPSV` — пассивный режим;
- `REST` — продолжение прерванной передачи;
- `MLSD` и `MLST` — машинный формат каталога;
- другие команды, которым сервер отвечает `502 Command not implemented`.

Сервер не объявляет `MLSD`/`MLST` в ответе `FEAT`, поэтому совместимый клиент должен использовать `LIST`, `NLST`, `SIZE` и `MDTM`. Клиент, требующий только `MLSD`, с этой версией несовместим.

## Ограничения

### Размер передаваемых файлов

| Направление | Действие пользователя | Ограничение плагина |
| --- | --- | --- |
| С компьютера на SD-карту ZX | загрузка файла на сервер | не более 256 КиБ для каждого файла |
| С SD-карты ZX на компьютер | скачивание файла с сервера | ограничения 256 КиБ нет |

Ограничение относится только к размеру одного файла, загружаемого с компьютера на ZX. После завершения передачи можно загружать следующие файлы, каждый размером до 256 КиБ. Причина в том, что перед записью на SD плагин целиком размещает входящий файл во внутреннем буфере из шестнадцати страниц по 16 КиБ.

При скачивании в обратном направлении плагин читает SD-карту небольшими блоками и сразу передаёт их компьютеру, поэтому размер скачиваемого файла внутренним буфером не ограничен.

### Другие ограничения

- одновременно обслуживается один управляющий FTP-клиент и одно data-соединение;
- второй управляющий канал получает `421 Only one FTP session is allowed`;
- один компонент пути ограничен 63 символами;
- полный путь ограничен 255 символами;
- FTP не шифрует имя пользователя, пароль и данные;
- сервер предназначен только для доверенной локальной сети.

## Диагностика

### Вход работает, но список каталогов не открывается

Типичный ответ сервера:

```text
425 Cannot open active data connection
```

Проверьте:

- выбран ли активный режим в профиле FileZilla;
- разрешён ли входящий TCP-диапазон FileZilla в брандмауэре;
- передаёт ли FileZilla локальный адрес компьютера из той же сети, что и ZiFi;
- не блокирует ли обратное соединение гостевая изоляция Wi-Fi;
- не занят ли выбранный локальный порт другой программой.

### Сервер отвечает `425 Send PORT or EPRT first`

Клиент пытается начать передачу без настройки активного data-соединения. Отключите пассивный режим и выберите активный.

### Сервер отвечает `421 Only one FTP session is allowed`

FileZilla открыл дополнительное управляющее соединение. В менеджере сайтов ограничьте число одновременных соединений значением `1`, закройте прежнюю вкладку и подключитесь заново.

### Сервер отвечает `530`

Проверьте `ftp_user` и `ftp_password`. После изменения `/zifi/zifi.ini` перезапустите плагин.

### Сервер отвечает `452 Cannot write file to SD card`

Проверьте свободное место, состояние файловой системы, доступность выбранного тома и корректность имени файла.

### Сервер отвечает `552 Upload exceeds 256 KiB plugin buffer`

Вы пытаетесь загрузить с компьютера на SD-карту ZX файл размером более 256 КиБ. Уменьшите или разделите этот файл. На скачивание файлов с ZX на компьютер это ограничение не распространяется.

### Плагин не находит конфигурацию

Проверьте точный путь `/zifi/zifi.ini`, наличие обязательных ключей и то, что параметры находятся в первых 511 байтах. Для NeoGS перед запуском откройте нужный том на активной панели Wild Commander.

### IP остаётся `0.0.0.0`

Адрес ещё не выдан DHCP. Проверьте SSID, пароль, уровень сигнала и настройки DHCP маршрутизатора. Плагин понимает как новый ответ `+CIFSR:STAIP,"192.168.x.x"`, так и старый ответ с отдельной строкой IPv4; `0.0.0.0` считается неготовым адресом.

## Обновление и удаление

Для обновления остановите плагин клавишей `Esc`, замените `/WC/ZIFIFTP.WMF` файлом из нового релиза и снова запустите Wild Commander.

Для удаления:

1. остановите сервер;
2. удалите строку `ZIFIFTP.WMF` из секции `[PLUGINS]` файла `/WC/wc.ini`;
3. удалите `/WC/ZIFIFTP.WMF`;
4. при необходимости удалите `/zifi/zifi.ini`.

## Сборка из исходников

Нужен ассемблер `sjasmplus`. Скрипт ищет его в следующем порядке:

1. путь из переменной среды `SJASMPLUS`;
2. `C:\z80\zuma\sjasmplus.exe`;
3. соседний каталог `..\..\ZiFi\sjasmplus.exe`;
4. команда `sjasmplus.exe` из `PATH`.

Сборка:

```bat
build.bat
```

Результат:

```text
ZIFIFTP.WMF
```

Полная проверка со сборкой, модельными тестами протокола и статической проверкой WMF:

```bat
test.bat
```

## Работа с FAT API Wild Commander

Файловая часть использует только API Wild Commander:

- `STREAM` создаёт отдельные потоки корня SD и текущего каталога FTP;
- `FENTRY`, `GDIR`, `ADIR` и `FindNext` работают с каталогами;
- `GFILE` и `LOAD512` читают файлы;
- `MKfile` и `SAVE512` создают и записывают файлы;
- `MKdir`, `REName` и `DelFl` изменяют файловую систему.

Низкоуровневые инструкции `IN`, `OUT` и `INIR` используются только для обмена с ZiFi.

## Структура проекта

- `src/main.asm` — заголовок WMF и точка сборки;
- `src/ftp_server.asm` — жизненный цикл окна и сервера;
- `src/network.asm` — AT-команды ESP, TCP и разбор `+IPD`;
- `src/zifi_uart.asm` — порты ZiFi, пакетный приём и тайм-ауты;
- `src/config.asm` — чтение `/zifi/zifi.ini` через FAT-драйвер;
- `src/ftp_commands.asm` — команды FTP и файловые операции;
- `src/ui.asm` — пользовательский интерфейс плагина;
- `tools/verify_build.py` — статическая проверка исходников и WMF;
- `tests/test_protocol.py` — модельные тесты протокола;
- `build.bat` — сборка;
- `test.bat` — сборка и все проверки;
- `zifi.ini.example` — шаблон конфигурации.

Исходные комментарии и документация сохранены в UTF-8 и написаны по-русски.
